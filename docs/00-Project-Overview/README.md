## 🎯 关键技术点

### 技术栈
- **前端**：React 18 + TypeScript + Zustand
- **后端**：Node.js + Express + Clean Architecture
- **AI**：LangChain + LangGraph + 豆包 LLM
- **数据**：Redis + PostgreSQL
- **部署**：Docker + 环境配置

### 核心特性
1. ✅ **无登录安全系统**：基于设备指纹的用户识别
2. ✅ **流式传输**：SSE 流式输出 + 断点续传
3. ✅ **多智能体协作**：基于 LangGraph 的多 Agent 系统
4. ✅ **大文本处理**：分块上传 + 压缩 + 渐进式加载
5. ✅ **工具调用**：LLM 工具系统 + 防幻觉机制

### 架构特点
- **Clean Architecture**：分层设计，依赖倒置
- **领域驱动设计**：清晰的业务边界
- **渐进式重构**：从传统架构平滑迁移
- **可测试性**：高内聚低耦合的模块设计

## 💡 面试要点

### 1. 架构设计
**问题：为什么选择 Clean Architecture？**
- **依赖倒置**：业务逻辑不依赖框架和数据库
- **可测试性**：核心业务逻辑可以独立测试
- **可维护性**：清晰的分层，易于理解和修改
- **可扩展性**：新增功能不影响现有代码

### 2. 技术选型
**问题：如何进行技术选型？**
- **需求分析**：项目需要什么功能？
- **方案对比**：考虑了哪些替代方案？
- **权衡取舍**：性能 vs 开发效率 vs 学习成本
- **实际验证**：通过 POC 验证可行性

### 3. 项目亮点
**问题：项目的核心创新点是什么？**
- **无登录安全**：在无登录情况下保护用户隐私
- **流式断点续传**：解决长时间流式传输的稳定性问题
- **大文本优化**：突破浏览器内存限制，处理超大文本
- **多智能体协作**：实现复杂的 AI 任务编排

### 4. 架构演进
**问题：如何进行渐进式重构？**
- **第一阶段**：清理冗余代码，建立清晰的模块边界
- **第二阶段**：引入 Clean Architecture，重构核心模块
- **第三阶段**：完善依赖注入，提升可测试性
- **持续改进**：根据反馈不断优化
